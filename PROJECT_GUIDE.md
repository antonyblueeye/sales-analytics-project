# 📘 Гид по проекту Sales CRM & Analytics

Этот файл — твоя главная инструкция. Здесь описано, как всё устроено, и что делать, если данные "решили прогуляться".

---

## 🏗 Структура проекта

### 📂 Backend (Папка `/backend`)
Твоя серверная часть на Python (FastAPI).

*   **`main.py`** — "Сердце" проекта. Главный файл, который запускает сервер и объединяет все части.
*   **`models.py`** — Описание таблиц в базе данных (Лиды, Кампании, Действия).
*   **`database.py`** — Настройки подключения к PostgreSQL.
*   **`analytics.py`** — Математика и SQL-запросы для графиков и таблиц дашборда.
*   **`crm.py`** — Логика работы CRM: список лидов, фильтры и сохранение твоих MQL/Call.
*   **`scheduler.py`** — "Мотор". Планировщик, который каждый час скачивает новые данные из MeetAlfred и делает бэкап.
*   **`crud.py`** — Вспомогательные функции для записи данных в базу.
*   **`backup_db.py`** — Скрипт для создания резервной копии базы.
*   **`restore_db.py`** — Скрипт для восстановления базы из копии.
*   **`rebuild_templates.py`** — Скрипт для полной пересборки шаблонов в Campaigns (с защитой).
*   **`cleanup_leads.py`** — Инструмент для удаления дубликатов лидов.
*   **`manual_actions.log`** — **Твой "Черный ящик"**. Текстовый файл, куда дублируются все MQL и Call, которые ты ставишь вручную.

### 📂 Frontend (Папка `/frontend-clean`)
Интерфейс на React/Next.js.

*   **`app/page.tsx`** — Главная страница дашборда с графиками.
*   **`app/crm/page.tsx`** — Страница CRM со списком лидов.
*   **`app/campaigns/page.tsx`** — Страница аналитики сообщений и шаблонов.
*   **`app/components/`** — Папка с кирпичиками интерфейса (таблицы, фильтры, кнопки).

## 🛡 Безопасность и Бэкапы

### 1. Как сделать бэкап вручную
Если ты сделал много важной работы и хочешь застраховаться прямо сейчас:
1. Открой терминал в папке `backend`.
2. Введи: `python backup_db.py`
3. Файл появится в папке `backend/backups/`.

### 2. Как восстановить данные (Если база пустая)
Если после перезагрузки ты зашел и видишь "пустоту":
1. Зайди в папку `backend/backups/`.
2. Выбери самый свежий файл (смотри по дате в названии).
3. В терминале введи:
   ```bash
   python restore_db.py "backups/имя_выбранного_файла.sql"
   ```
4. На вопрос "Are you sure?" введи `yes`.
5. Обнови страницу в браузере — всё вернется.

## 🆘 Экстренные ситуации

### "Я поставил MQL, но его нет в базе"
1. Открой файл `backend/manual_actions.log`.
2. Если запись там есть — значит, данные в безопасности, просто произошел глюк отображения.
3. Если записи нет и в логе — значит, произошла ошибка при сохранении (проверь интернет или не упал ли сервер).

### "Дашборд показывает нули"
1. Проверь даты в фильтре (справа сверху). Теперь там по умолчанию "Последние 7 дней", но ты всегда можешь выбрать другой период.
2. Проверь, запущен ли сервер (`uvicorn main:app --reload`).

## 🚀 Как правильно запускать проект после перезагрузки

1.  **Запусти PostgreSQL** (обычно он запускается сам как служба Windows).
2.  **Запусти Backend**:
    *   Зайди в `backend`.
    *   `venv\Scripts\activate` (активируй окружение).
    *   `uvicorn main:app --reload`.
3.  **Запусти Frontend**:
    *   Зайди в `frontend-clean`.
    *   `npm run dev`.
4.  Открой в браузере `http://localhost:3000`.
